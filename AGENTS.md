## SEO & Copy Standards

Every agent must follow these writing rules for all metadata, copy, alt text, and schema fields.

### The Core Rule

Write as a knowledgeable human who respects the reader's time. If the output sounds like it
was generated by an AI, rewrite it.

### Metadata

**Title tags**
- Format: `[Vehicle or Page Subject] — Planet Motors Richmond Hill`
- Lead with the strongest keyword, follow with brand
- Max 60 characters
- Must be unique per page

**Meta descriptions**
- One or two sentences. Include one specific, honest detail about the page.
- End with a natural soft CTA (not "Click here" or "Don't hesitate")
- Max 155 characters
- Must be unique per page

**Canonical URLs**
- Always point to `/inventory/used/[make]/[model]/[slug]` for VDPs
- Never emit `/inventory/[slug]` as canonical in any metadata or sitemap

### Vehicle Copy

Lead with a specific, honest observation about the vehicle:
```
Good: "This 2019 Mazda CX-5 GT came in as a one-owner trade with 61,000 km and a clean Carfax."
Bad:  "Explore our quality pre-owned Mazda inventory at Planet Motors."
```

Never invent vehicle facts. If detail is unknown, omit it.

### Banned Phrases

Never use in any output:
- dive into, delve into, let's explore, it's worth noting
- seamless, robust, cutting-edge, game-changer, leverage (verb)
- in today's world, in today's digital age
- a testament to, look no further, your one-stop shop
- we pride ourselves on
- Welcome to Planet Motors (never open a page with this)
- Don't hesitate to contact us

### Alt Text

Must be specific and descriptive:
```
Good: "Front three-quarter view of a 2019 white Mazda CX-5 GT at Planet Motors"
Bad:  "Used SUV" or "vehicle-image-1"
```

### JSON-LD / Structured Data

- `description` fields: full readable sentences, not keyword strings
- `name` fields: real entity names, not SEO phrases
- Never fabricate schema values — omit unknown fields rather than guess

## Agent Behavior Rules

- **Never generate placeholder copy.** Write real, usable draft content every time. If you
  lack information to write real copy, state what you need rather than filling in with generic text.
- **Never pad.** Say what needs to be said and stop.
- **Always run before considering a task done:**
  ```bash
  npm run lint && npm run typecheck && npm run build
  ```
- **Never touch:** Clerk, Stripe, AutoRaptor, RouteOne, Dealertrack — those are A5 scope.
- **Never redesign UI components** — A4 is performance, SEO, and media only.
- **Never change route policy** — A1 route policy is locked.

## Definition of Done (A4)

A task is complete only when all of the following are true:

- [ ] Metadata strategy is aligned to the canonical nested route policy
- [ ] Sitemap emits canonical nested VDP URLs only — short routes excluded
- [ ] Robots.txt allows all public routes correctly
- [ ] Cloudinary image/runtime rules are defined and applied safely
- [ ] 360 runtime behavior is explicitly isolated from critical render path
- [ ] Homepage, inventory, and VDP have a performance hardening plan
- [ ] LCP and CLS risks are identified and addressed
- [ ] JSON-LD structured data is implemented and validated
- [ ] Cache and revalidation strategy is documented and stable
- [ ] `npm run lint`, `npm run typecheck`, and `npm run build` all pass
